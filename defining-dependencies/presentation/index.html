Define Your Dependencies

2 ways to track requirements

Micah Denbraver

SDPUG, October 22, 2015

Use requirements.txt

  1. Install your packages as needed during development
    pip install awesome-print  # or some other awesome package
  2. Save periodic snapshots of the current environment
    pip freeze > requirements.txt
    For our Windows friends:
    pip freeze | Out-File requirements.txt
  3. Install previously recorded requirements
    pip install -r requirements.txt

Use setup.py

Ideal for reusable libraries or installable tools

setup(
    name='coolpackage',
    version='0.1.0',
    author='Yours Truly',
    author_email='yt@example.com',
    packages=find_packages(),
    description='Your cool package.',
    install_requires=[
        'awesome-print',
        'requests>=2,<3',
    ],
)

Loose package requirements, better for being combined with other packages in the same environment.

setup.py vs requirements.txt

tl;drJust use requirements.txt

  • When it matters, and you've decided to make your package public, the dependencies will need to be reviewed anyway.
  • They are not the same, don't load requirements.txt somehow into setup.py.
